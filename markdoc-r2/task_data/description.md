# 任务 Prompt

为 Flask-RESTful 实现 API 版本控制机制，具体需求如下：
1. 在 Api 类中添加版本控制支持，允许通过 URL 路径（/v1/、/v2/）或请求头（Accept: application/vnd.api+json;version=1）指定版本
2. 实现版本路由装饰器，允许同一 Resource 类在不同版本下有不同的行为
3. 支持版本继承，新版本可以继承旧版本的资源定义，只需重写变化的部分
4. 实现版本弃用机制，当访问已弃用版本时返回警告头（Deprecation）和迁移指南
5. 支持版本别名，允许 'latest'、'stable' 等别名指向具体版本
6. 实现版本比较和范围匹配（如 version>=1.0,<2.0）
7. 添加版本信息到 API 响应中，便于客户端识别当前版本
8. 支持 Blueprint 级别的版本控制，允许不同蓝图使用不同版本策略
9. 提供版本迁移工具，帮助开发者从旧版本迁移到新版本
10. 编写完整的文档和示例，说明版本控制的最佳实践

## 题目方向
业务逻辑

Hard: 极致性能/资源优化（版本路由性能）
