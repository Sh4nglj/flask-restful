# 任务 Prompt

修复 Flask-RESTful 错误处理机制中的内存泄漏问题，具体需求如下：
1. 修复 handle_error 方法中异常对象和堆栈信息未正确释放的问题
2. 修复当 errors 字典包含大量自定义错误定义时，内存持续增长的问题
3. 修复当异常包含循环引用时，异常对象无法被垃圾回收的问题
4. 修复 exc_info 三元组在某些情况下未正确清理，导致内存泄漏
5. 优化错误响应对象的创建，避免不必要的对象复制和内存分配
6. 修复当自定义错误处理器抛出异常时，异常链未正确清理的问题
7. 实现错误对象的弱引用机制，避免错误对象长期占用内存
8. 修复当使用 catch_all_404s=True 时，404 错误对象累积的问题
9. 添加内存使用监控，检测错误处理过程中的内存泄漏
10. 编写压力测试，验证长时间运行下内存使用稳定

## 题目方向
运行时异常修复

## 代码量估算
Hard: ≥300行多文件

## 技术栈广度
Hard: 跨技术栈≥3种（Python、内存管理、性能分析工具）

## 优化要求
Hard: 极致性能/资源优化（内存使用优化）

## 人工开发耗时预估
Hard: ≥4小时
