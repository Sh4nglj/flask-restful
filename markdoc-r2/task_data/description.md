# 任务 Prompt

为 Flask-RESTful 实现请求限流中间件功能，具体需求如下：
1. 在 flask_restful/utils 目录下创建 rate_limit.py 模块，实现基于令牌桶算法的限流器
2. 支持基于 IP 地址、用户 ID 或自定义键的限流策略
3. 实现装饰器 @rate_limit，可配置每秒请求数（RPS）和突发请求数（burst）
4. 支持全局限流和资源级别的限流配置
5. 当请求超过限流阈值时，返回 429 Too Many Requests 状态码，并在响应头中包含 Retry-After 信息
6. 支持多种存储后端（内存、Redis），允许分布式环境下的限流
7. 实现滑动窗口算法作为备选限流策略
8. 添加限流统计功能，记录被限流的请求数量和频率
9. 提供配置选项，允许白名单 IP 或用户绕过限流
10. 编写完整的单元测试和集成测试，确保限流功能正确工作
