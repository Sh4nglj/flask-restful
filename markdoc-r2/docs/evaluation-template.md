# 评估模板

## 说明

* 运行项目评价代码的可行性。如报错，分析报错原因并写入文档。

* 可根据需要运行项目或构建单元测试进行评估。

* 不输出emoji。

## 字段



## 格式

### 评价内容格式（存入 content.json）

使用纯文本格式，不使用 markdown 语法，以交付完整性为例：

```
已实现需求：\n1. 功能点A完全实现\n2. 功能点B符合要求\n\n未完整实现或存在严重问题：\n1. 缺少单元测试\n2. 数据库脚本不匹配\n - name.java和name.sql字段冲突 \n 3. 编译失败 \n - name.java 存在报错 \n评分理由：根据打分标准给予3分。
```

**注意**：
- 150字以内。
- 错误部分列举出具体的
- 不使用任何 markdown 格式（如 `**`、`#`、`` ` ``）
- 换行使用 `\n` 表示
